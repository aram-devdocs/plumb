# gh-runbook default workstream topology

Use grouped workstreams for audit-driven release planning.

## Parent issue

- release readiness runbook

## Child workstreams

1. trading and export correctness
2. trading safety and order-entry gating
3. dead-letter queue and schema incident resolution
4. admin orchestration health or hide
5. onboarding, help, tutorial, and changelog consolidation
6. feedback, support, and bug-report consolidation
7. dashboard and analytics scope rationalization
8. auth invite and token fallback states
9. Storybook runtime gate and page-state coverage
10. launch-scope cleanup and admin information architecture follow-up

## Required fields per issue

- summary
- acceptance criteria
- dependencies
- effort estimate
- implementation notes

Implementation notes should also include:

- suggested package or layer scope
- related existing issues or PRs
- suggested delivery skill or reviewer
